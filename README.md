# 日常工具集 🛠️

一个基于React构建的现代化日常工具集Web应用，集成开发工具、天气预报、日历黄历和新闻聚合功能。

[![React](https://img.shields.io/badge/React-18.3.1-blue.svg)](https://reactjs.org/)
[![Vite](https://img.shields.io/badge/Vite-6.0.1-646CFF.svg)](https://vitejs.dev/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

**工具数量**: 4个主工具 + 3个开发子工具 = 7个实用工具

## ✨ 功能特性

### 💻 Coding工具（新增）

#### 🔢 计算器
- **标准模式**：基础四则运算
- **程序员模式**：
  - 四种进制转换（HEX/DEC/OCT/BIN）实时显示
  - 六种位运算（AND/OR/XOR/NOT/<</>>）
  - 十六进制字母输入（A-F）
  - 横向布局，一屏完整显示

#### 📝 字符串处理
- **大小写转换**：大写、小写、首字母大写
- **字符串操作**：反转、去空格、删除所有空格
- **编码解码**：Base64、URL编码/解码
- **HTML转义**：防止XSS攻击
- **统计功能**：字符数、单词数、行数统计
- **一键复制**：处理结果快速复制

#### { } JSON工具
- **实时校验**：边输入边检查JSON格式
- **格式化**：自动美化JSON结构
- **压缩**：移除空格和换行
- **详细错误**：精确显示错误位置和原因
- **示例加载**：快速加载示例JSON
- **一键复制**：复制格式化结果

### 🌦️ 天气工具

- **当前天气**：实时温度、天气状况、湿度、风力、降雨概率、温度范围
- **48小时预报**：逐小时温度曲线图和降雨概率曲线，1小时间隔，支持横向滚动
- **15天预报**：未来两周天气趋势，包含最高/最低温度、风力、降雨概率
- **城市切换**：支持手动输入城市名称查询天气

### 📅 日历工具

- **月视图日历**：清晰的月份日历展示，可点击日期
- **农历信息**：显示农历日期、节日、二十四节气（金色醒目标签）
- **节假日标注**：2024-2025年中国法定节假日高亮显示
- **调休标识**：标注调休上班日期
- **月份切换**：便捷的上月/下月/今天导航
- **黄历面板**：
  - 干支纪年（年月日天干地支）
  - 宜忌事项（绿红双栏显示）
  - 冲煞、五行、胎神信息
  - 吉神凶煞详情

### 📰 新闻工具（新增）

- **热点聚合**：来自TrendRadar的多平台新闻热点
- **自动加载**：页面打开自动获取最新内容
- **手动刷新**：一键更新新闻内容
- **时间显示**：显示上次刷新时间
- **完整展示**：保留原页面所有功能和样式

## 🚀 快速开始

### 环境要求

- Node.js 16.0+
- npm 7.0+ 或 pnpm

### 安装依赖

```bash
npm install
```

### 开发模式

```bash
npm run dev
```

应用将在 `http://localhost:3000` 启动。

### 生产构建

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

### 预览生产构建

```bash
npm run preview
```

### 代码检查

```bash
npm run lint
```

## 🎨 界面预览

### 主界面布局
- 左侧：竖向工具切换Tab（Coding、天气、日历、新闻）
- 右侧：工具内容展示区
- 支持响应式设计，移动端自适应

### Coding工具
- 横向子Tab切换（计算器、字符串处理、JSON校验）
- 程序员计算器：进制显示与按钮横向并排
- 字符串处理：三栏布局（输入、操作、输出）
- JSON工具：双栏布局，实时校验

### 天气工具
- 渐变背景，卡片式设计
- SVG绘制的温度和降雨曲线
- 48小时预报支持横向滚动
- 现代化的交互体验

### 日历工具
- 网格布局，清晰易读
- 颜色区分节假日和调休
- 农历信息小字显示，节气金色标签
- 点击日期查看详细黄历

### 新闻工具
- 完整嵌入TrendRadar新闻页面
- 刷新按钮和时间显示
- 加载状态和错误处理

## 🔧 开发规范

### 代码风格

- 组件：使用函数式组件 + Hooks
- 命名：驼峰命名法（组件大驼峰，变量小驼峰）
- 注释：关键逻辑添加英文注释
- 格式：通过ESLint检查

### Git提交规范

遵循 Conventional Commits 规范：

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具链相关
```
